# CharacterArchive

## Project's Purpose
First, a little background.  I've been playing DnD for almost a year.  It's been a lot of fun.  Unfortunately, I don't really have a lot of time to dedicate in building amazing characters.  I love stories, and I'm a sucker for character-led games.  I thought this would be a great time to combine my hobby and career training into one reasonably "simple" project.  Of, couse we all know if you think a project should be "simple," it is likely to balloon out of control!  As such, I want to list out some objectives that this project should focus on.

1. Build a React application that clearly shows a DnD's character sheet in a presentation that fits the "moment."  For example, what do I most use when we are in the middle of conflict?  What would I like to look at when I'm trying to solve a mystery?  How should I best recall what we did last week?
2. I want to re-engage tech that I've used or trained on.  As such, I will be building this project in phases.  Each phase will be focused on specific tech.  (1) <u>React</u> for the front-end, (2) <u>PostgresSQL</u> for data management, (3) <u>Java</u> for CLI backend data input, (4) <u>cloud</u> deployment application, (5) <u>GoLang</u> refactor CLI backend application, (6) <u>Fyne</u> for backend GUI application, (7) <u>React</u> for backend Web application, (8) <u>Iterate</u>
3. Provide the character building community a place where they can organize, share, and gather character-driven story ideas for TTRPGs or other such fictional writing media outlets.  This will also require phases that need to be iteratively introduced into the application.  (1) Character Sheet Display, (2) Multiple characters, (3) Multiple user access, (4) Multiple character build types from different TTRPGs, (5) Community news page, (6) Community share page, (7) Community rating and discussion forum
